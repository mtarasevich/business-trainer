# Требования к сайту

## Тип сайта
Одностраничный лендинг-визитка эксперта (single page application с якорными ссылками). Цель — конвертировать посетителя в заявку на тренинг / коучинг / выступление.

## Целевые действия (приоритет)
1. **Заявка на корпоративный тренинг** (главный CTA)
2. Запись на индивидуальный коучинг
3. Приглашение на выступление / keynote

Все три ведут на одну форму с предзаполненным полем «тип запроса».

## Функциональные требования
- **Hero**: фото эксперта, имя, заголовок (роль), 1 строчка позиционирования, 2 CTA-кнопки (главный + вторичный).
- **О себе**: текст 2–3 абзаца, фото, цифры-результаты (4–6 метрик карточками), таймлайн опыта/образования.
- **Услуги**: 4 карточки (топ-тренинги) + расширяемый список форматов работы. Каждая карточка: название, краткое описание, "для кого", CTA "Обсудить".
- **Экспертиза**: облако/чипы тем + сегменты рынка (визуально).
- **Кейсы / результаты**: 1–3 кейс-блока с цифрами (главный кейс + рост оборота + рост клиентской базы).
- **Видео**: встроенный YouTube-плеер с подкастом + плеер с локальным видео из inputs.
- **Фотогалерея**: компактная сетка 6–9 фото с тренингов (lightbox при клике).
- **Образование / сертификаты**: список или таймлайн.
- **Отзывы** — нет данных, опускаем (см. ASSUMPTIONS).
- **Контакты**: email, телефон, мессенджеры (кликабельные ссылки tg://, viber://), форма обратной связи.
- **Форма**: имя, контакт (email/телефон), тип запроса (select), сообщение. Валидация. Submit — на email клиента (mailto fallback или formspree-like; для статичного билда — mailto).

## Нефункциональные
- Полностью статичный HTML/CSS/JS (без бэкенда).
- Адаптивный дизайн: mobile-first, breakpoints 768/1024/1440.
- Производительность: фото оптимизированы (≤300КБ each, lazy-load), видео — preview poster + click-to-play.
- SEO-минимум: meta title/description, OG-теги, semantic HTML, alt на всех картинках.
- Доступность: контраст AA, фокус-стили, keyboard navigation, aria-labels на иконках.
- Браузеры: последние Chrome/Firefox/Safari/Edge.
- Язык интерфейса: русский. Без многоязычности.

## Стек
- HTML5 + CSS (vanilla, custom properties для темы) + JS (vanilla, без фреймворков).
- Шрифты: 1 пара (heading + body) с Google Fonts или локально.
- Иконки: inline SVG.
- Фото: оригиналы из `inputs/` ужать в `05-build/assets/img/` (jpg, ~1600px max width, q=80).

## Out of scope (на будущее)
- CMS / админка — нет.
- Блог-движок — нет (раздел "Блог/статьи" пока заглушка-секция "Скоро" или вообще убрать — см. ASSUMPTIONS).
- Личный кабинет, оплата онлайн — нет.
- Многоязычность — нет.
- Аналитика (GA / Метрика) — слот в `<head>` подготовить, ID не вставлять.
